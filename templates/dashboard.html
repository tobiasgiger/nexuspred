    Getting started
    
        
    
          This guide walks you from a fresh machine to live signal routing. Work
          through the steps in order. New here? Stay on a Demo
          account until you've seen test signals fill correctly.
        
    
        
    
          1 · Install
          2 · Run
          3 · Tradovate API
          4 · Connect
          5 · Trading rules
          6 · TradingView alert
          7 · Go live
          8 · Updates
        
    
      
    

      
    
        
    1 · Install
    
        
    Pick your operating system. Each installer creates an
          isolated environment and a one-click launcher — nothing else on your
          system is touched.
    
        
    
          
    
            
    🐧 Linux / macOS
    
            
      
              
    1. 1
      Download or clone the project, then open a terminal in its folder.
      2. 
              
    2. 2
      Run the installer:
                
      chmod +x install.sh
./install.sh
      
                Optional: ./install.sh --service auto-starts the bridge on boot (systemd), --port 9000 changes the port.
              
      3. 
            
    
          
    
          
    
            
    🪟 Windows
    
            
      
              
    1. 1
      Install Python 3.9+ and tick “Add python.exe to PATH”.
      2. 
              
    2. 2
      Double-click install.bat (or run it in a terminal). It builds the environment and creates start.bat.
      3. 
            
    
          
    
        
    
      
    

      
    
        
    2 · Run the bridge
    
        
      
          
    1. 1
      Start it:
            
      Linux/macOS:  ./start.sh
Windows:      start.bat
      2. 
          
    2. 2
      Open http://localhost:9000 in your browser — you're looking at the dashboard right now.
      3. 
        
    
        
    💡 The bridge must keep running to receive signals. On Linux use --service; on Windows leave the start.bat window open or add it to startup.
    
      
    

      
    
        
    3 · Choose how to authenticate
    
        
    Two ways to connect (a token is always used first if available):
    
        
      
          
    1. 1
      Username & password (easiest) — just your Tradovate login. With “web-trader fallback” on, it connects without a paid API subscription. Have an API key? Add cid/sec under Advanced.
      2. 
          
    2. 2
      Access token — set the TRADOVATE_ACCESS_TOKEN (and optional TRADOVATE_CHECK_TOKEN) environment variables, or paste them under “Use an existing access token”. The token is used first and auto-renewed (access → check token), falling back to a credentials login.
      3. 
        
    
        
    ⚠ Start with the Demo environment. Switch to Live only after verifying fills.
    
      
    

      
    
        
    4 · Connect
    
        
      
          
    1. 1
      Open the Settings tab and fill in the Tradovate Connection card. Leave Environment on Demo.
      2. 
          
    2. 2
      Click Save Settings, then Connect & Verify. Your account is auto-detected and the header turns green.
      3. 
        
    
      
    

      
    
        
    5 · Confirm trading rules
    
        
    Defaults match the strategy in the screenshots. Adjust on the Settings tab if needed.
    
        
    
          
    SignalOrderTypeQty
    
          
            
    buy / sellEntryMarket3 (default)
    
            
    tp1 / tp2 / tp3Take-profitLimit1 each
    
            
    slStop-lossStopfull position
    
            
    move_slMove the stop——
    
            
    close_allCancel + flattenMarketfull position
    
              
        
    
        
    🔒 Trading enabled is OFF by default — a safety switch. Turn it on only when you're ready to send real orders.
    
      
    

      
    
        
    6 · Point TradingView at the bridge
    
        
      
          
    1. 1
      In Settings, set a strong Webhook secret and copy your URL from the Test & Webhook tab. It looks like:
            
      http://YOUR_HOST:9000/webhook/YOUR_SECRET
      
            For TradingView to reach it, the bridge must be on a public address (port-forward, VPS, or a tunnel like ngrok / Cloudflare Tunnel).
          
      2. 
          
    2. 2
      Create a TradingView alert → Notifications → enable Webhook URL and paste the URL above.
      3. 
          
    3. 3
      Set the alert message to the strategy JSON, e.g.:
            
      {"event":"entry","action":"sell","symbol":"MNQ1!",
 "entry":30267,"sl":30285.07,
 "tp1":30261.58,"tp2":30265.19,"tp3":30247.04}
      4. 
          
    4. 4
      Before going live, open Test & Webhook, load a preset and click Send Test Signal to watch it flow through.
      5. 
        
    
      
    

      
    
        
    7 · Go live
    
        
      
          
    1. 1
      Verify demo fills on the Monitor tab (positions, orders) and the Logs tab.
      2. 
          
    2. 2
      In Settings, switch Environment to Live, re-enter live credentials, Connect & Verify again.
      3. 
          
    3. 3
      Flip Trading enabled on. You're live. 🚀
      4. 
        
    
      
    

      
    
        
    8 · Keep it updated
    
        
    The bridge checks GitHub for new versions automatically. When one is available, an amber Update available button appears in the header — click it to pull the latest code and restart. You can also check on demand from the Updates card in Settings.
    
        
    🛟 Trouble? The Logs tab shows every signal and event with timestamps — the fastest way to see what happened.
